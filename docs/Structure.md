# Структура проекта

Корневая структура

```
├───data
├───db
├───docs
├───logs
└───src
```

`data/` - Различные данные

`db/` - Локальная база данных, то есть если у нас не подключен Postgresql

`docs/` - Документация проекта

`logs/` - Хранилище логов

`src/` - Бизнес-логика и архитектура бота

```
└───src
    ├───config
    ├───core
    ├───filters
    ├───handlers
    ├───middlewares
    ├───models
    ├───services
    └───utils
```

`config/` - Конфигурации: токены, ключи, пути, настройки базы.

`core/` - Инициализации приложения (использован app_context для удобной работы с зависимостями)

`filters/` - Пользовательские фильтры aiogram (например, IsAdmin)

`handlers/` - Все хендлеры: команды, сообщения, колбэки

`middlewares/` - Middleware для логирования, rate-limit и т.д.

`models/` - Модели базы данных

`services/` - Сервисы

`utils/` - Разные вспомогательные функции, (например, генерация уникального кода товара)
